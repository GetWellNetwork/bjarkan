bjarkan.logger module¶

GWN logging module. Handles crafting messages to be placed into the system journal.

Examples

from gwn.helpers.logger import logger
# ...
logger.info( 'starting up main loop' )

Notice that str.format() is handled intenrally for you:

logger.debug( 'found {:d} files', len( my_files ) )

Setting the Log Level¶

You can override the default log level (INFO) via the GWN_LOGGER environment variable

GWN_LOGGER=DEBUG /usr/bin/gwn-foo

It may be set to any of the following values (case insensitive):

  • critical
  • error
  • warning
  • info
  • debug

Note that most of our services include at least two EnvironmentFile= declarations in their unit file designed to help you easily instrument this mechanism. For example, from the registration service you will see the following snippet.

[Service]
EnvironmentFile=-/etc/default/gwn
EnvironmentFile=-/etc/default/gwn-registration

So you can put GWN_LOGGER="debug" in either that gwn file or that gwn-registration file to turn on debug logging for all of our services or just for registration, respectively.

echo 'GWN_LOGGER="debug"' | sudo tee /etc/default/gwn-startup-config

To determine what environment files a particular service will read, look at its unit file. The easiest way to do that is to use the systemctl cat command:

systemctl cat gwn-support-bridge.service

class bjarkan.logger.StyleAdapter(logger, extra=None)¶

Bases: logging.LoggerAdapter

A wrapper around a logging.Logger object that adds str.format() capabilities so you can do things like:

logger.debug( 'found {} records', len( my_records ) )

without calling format() directly and without incurring the cost of calling it if the current log level would prevent this statement from actually being logged.

When you from gwn.helpers.logger import logger, this is the object you get.

log(level, msg, *args, **kwargs)¶

Implements the str.format() logic. You probably shouldn’t be calling this method directly.

setFormat(fmt, style='{')¶

A convenience wrapper so you don’t have to instantiate a logging.Formatter object to override the log statement format.

See the standard library docs for more details. Generally, you won’t need to call this method.

getLevel()¶

A convenience wrapper around getEffectiveLevel() because the integer values for the various logging levels are clunky and probably don’t mean anything to you.

Returns:str – the name of the effective log level for this logging object, in lowercase ("warning", "info", etc.)
setLevel(lvl)¶

A convenience wrapper around the super method of the same name, because upper case sucks.

Parameters:lvl (str) – a standard logging level name ("warning", "debug", etc.); will be cast to upper case for you, since the super method is strict about that

Notes

  • the super method also accepts integer values from the logging.LEVEL_NAME constants, those will be passed through, even though the above docs state that lvl should be a str.
duplicateToStderr()¶

Add a handler to the logger that also sends messages to stderr in addition to the journal. After called the first time, subsequent calls to this method will do nothing.
